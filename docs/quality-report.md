## 5. 已知风险与技术债务

### 5.1 高风险

| 风险项 | 影响 | 缓解措施 |
|--------|------|---------|
| AI 服务无降级策略 | API 超时（如 SEO 5min）用户仅看到 score 0 | 前端增加 score 为 0 时的提示文案；或后端区分 fallback 响应码 |
| **覆盖率大幅回落** | 新增代码无测试，回归风险增加 | **优先级高：为 optimizeSEO 及新增 DTO 补充单元测试** |

### 5.2 中风险

| 风险项 | 影响 | 缓解措施 |
|--------|------|---------|
| 前端覆盖率缺失 | 无法量化组件测试质量 | 安装 vitest coverage 插件并补测 |
| E2E 测试仅 16 条 | 用户交互场景覆盖不足 | 增加关键路径 E2E 用例 |
| AI 生成内容无 watermark | 难以区分人工与 AI 内容 | 建议产品层面增加 AI 生成标识 |

### 5.3 技术债务

1. **覆盖率缺口扩大**：AI SEO 功能约 150 行新增代码零测试覆盖，需优先补齐。
2. **类型安全**：部分 DTO 使用 `any` 类型（如 `parsed` 变量），建议增加 Zod 或 class-validator 校验。
3. **错误日志**：`Logger.error` 在测试中被 mock，生产环境需确保日志链路完整。

## 8. 质量结论与建议

### 8.1 总体评价

**质量等级：B+** ⚠️

本轮迭代在快速推进 AI SEO 功能和 RSS 热点增强的同时，保持了 100% 测试通过率。但**新增代码零测试覆盖导致整体覆盖率显著下滑**（函数覆盖率从 94.66% 降至 82.41%），这是当前最突出的质量风险。

### 8.2 放行建议

- **合并至 main 前必须**：为 `optimizeSEO` 方法补充单元测试，将覆盖率拉回 85%+。
- **发布前建议**：
  1. 完成前端覆盖率测量与补齐
  2. 增加 E2E 关键路径用例（选题→稿件→审核→发布完整流程）
  3. 处理高风险项：AI fallback 的 UX 问题

### 8.3 下轮迭代 QA 关注

1. **补齐 AI SEO 单元测试**（优先级最高）
2. AI 生成内容的水印/标识需求验证
3. 性能测试（AI API 超时场景、大数据量稿件加载）
4. 权限矩阵全面回归（Reporter / Editor / Admin 边界场景）
