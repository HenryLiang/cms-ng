# 01 创作大脑（CMS-NG）产品介绍

> 版本：v1.0
> 适用对象：产品负责人、潜在客户、合作伙伴、业务决策者
> 文档定位：业务视角为主的产品概览，去除技术实现细节

---

## 0. 一句话定位

**01 创作大脑（CMS-NG）是一套面向媒体机构的 AI 驱动内容生产作业系统，以"记者 + AI"协作为核心，覆盖从热点发现、稿件创作、编辑审核到多平台分发的完整内容生产链路，并提供业内少见的"全自动内容管道"作为人力补充。**

---

## 1. 产品核心能力

CMS-NG 不是单一的"AI 写作工具"，而是一套**面向新闻编辑部的全栈式内容生产操作系统**。系统围绕"记者工作台"、"编辑工作台"、"AI 协作能力"、"多平台分发"和"全自动内容管道"五条主线展开，并配套"用户与权限"、"通知与运维"两大支撑模块。

### 1.1 记者工作台

记者是一线使用者，系统目标是让"找选题 → 写稿件 → 配图 → 提交审核"的链路尽可能顺滑。

- **选题中心**：支持手动录入、AI 智能推荐、12 路 RSS 源（含 Google Trends、BBC、纽约时报、联合早报、36 氪、虎嗅等）一键导入、24 小时去重；
- **稿件管理**：覆盖 9 个状态的全生命周期管理（草稿 → 采写中 → AI 优化中 → 待审核 → 审核中 → 退回修改 → 审核通过 → 已发布 → 已归档）；
- **版本快照与回滚**：每轮 AI 操作或人工修改都生成独立版本快照，支持任意历史版本一键回滚；
- **AI 配图**：基于文章标题与摘要自动生成提示词，调用 AI 图像模型产出封面图，并自动上传至对象存储；
- **个人专长与偏好**：记者可设置擅长领域与偏好的语言风格，AI 推荐时自动加权。

### 1.2 编辑工作台

编辑是稿件质量与发布风险的"守门人"。

- **审稿队列**：统一视图查看所有待审稿件，支持按记者、栏目、紧急程度排序；
- **分配编辑**：支持将稿件指派给指定编辑；
- **退回修改**：可附带修改意见，稿件回到修改状态后记者继续迭代；
- **AI 审稿报告**：从事实性、立场、敏感词、SEO、读者友好度等多维度生成审阅报告，作为编辑的"第二双眼睛"；
- **自动发布管理面板**：可查看、启停、手动触发自动发布任务，并实时监控每次运行的执行情况。

### 1.3 AI 协作能力

系统提供 **14 项 AI 原子能力**，覆盖写、读、审、辅四大类：

| 类别 | 能力 | 业务价值 |
|---|---|---|
| **写** | 改写 / 扩写 / 缩写 / 润色 / 多标题生成 / 摘要生成 / 对话写作 / 初稿生成 | 覆盖从一句话到完整稿件的全流程写作需求 |
| **读** | 资料调研（联网搜索）/ 事实核查（联网佐证）/ AI 选题推荐（基于热点 + 记者专长） | 替代大量人工搜索与调研时间 |
| **审** | AI 审稿（多维评分） | 辅助编辑交叉验证 |
| **辅** | SEO 优化 / AI 配图 | 提升内容可发现性与视觉吸引力 |

**两条不可妥协的原则**：
- 所有 AI 输出**默认需人工确认**才能进入发布环节；
- 所有 AI 操作**全部留痕**（含 prompt、模型、token 消耗、耗时、结果），形成可审计的 AI 使用轨迹。

### 1.4 多平台分发

系统目前已落地 **5 个真实可用的发布适配器**，每个适配器都拥有**独立的 prompt 模板与后处理流水线**：

| 平台 | 适配点 |
|---|---|
| **Website**（自有站） | 站内发布、栏目分类、SEO 字段映射 |
| **Facebook** | 标题 / 正文长度截断、Hashtag 风格化、配图比例 |
| **Instagram** | 首图强制 1:1、文案 emoji 化、配 9–30 个 hashtag |
| **小红书** | 标题党化、emoji 化、3–5 个话题标签、口语化表达 |
| **WordPress** | 完整 REST API 集成（含分类 / 标签 / 特色图） |

**业务价值**：同一篇稿件在不同平台呈现完全不同的标题、长度与语气；后端为**每平台保存一份独立版本**，避免平台间相互污染——一篇稿件 1 键分发到 5 个平台只需 30 秒。

### 1.5 全自动内容管道（核心差异化能力）

这是本产品区别于市面上绝大多数"AI 写作工具"的**关键差异化模块**——它提供一条**完全无需人工干预**的内容生产流水线。

**管道执行的 8 个阶段**：

| 阶段 | 动作 |
|---|---|
| 1. 调度触发 | 支持指定时间点 / 周期 / cron 表达式三种调度 |
| 2. 选题采集 | 从热点源 + 固定栏目关键词收集候选 → 敏感词过滤 → 24h 去重 → 选出 N 个 |
| 3. 资料调研 | 联网搜索收集背景资料与数据 |
| 4. 文章生成 | 按任务级风格 prompt 生成标题 / 正文 / 摘要 |
| 5. 封面配图 | 自动生成封面图并存储 |
| 6. 保存文稿 | 创建 Story + Article，状态直接置为"已自动发布"（**跳过人工审核**） |
| 7. 平台发布 | 通过 WordPress 适配器推送 |
| 8. 通知汇报 | 邮件汇总本次成功 / 失败明细 |

**四道安全兜底**：
- **半成品保存**：失败步骤的中间产物会落库，不丢失任何已完成的工作；
- **单篇重试 / 撤回**：失败文章可单独重试或从平台撤回；
- **全局 Kill Switch**：一个操作即可停止所有自动发布任务（防失控兜底）；
- **敏感词过滤 + 邮件告警**：在选题阶段就过滤高风险领域，任务结束后立即汇总通知。

### 1.6 用户与权限

采用经典的三级角色权限模型：

| 角色 | 权限范围 |
|---|---|
| **记者** | 创建选题、撰写稿件、使用 AI 工具、提交审核 |
| **编辑** | 审核稿件、分配选题、管理发布流程、查看审稿队列 |
| **管理员** | 用户与角色管理、系统配置、查看全部数据 |

### 1.7 通知与运维

- **邮件通知**：稿件审核流转、自动发布成功 / 失败、关键异常事件触发邮件；
- **审计日志**：所有 AI 调用全程留痕，可用于复盘与合规审计；
- **一键部署脚本**：封装"备份 → 拉代码 → 构建 → 迁移 → 重启 → 探活"完整链路，单条命令即可完成生产更新。

## 2. 创新点：与同类产品的差异化优势

CMS-NG 在业务模式、产品体验与部署灵活性上均做了独立思考，并非简单复刻市面上已有的 AI 写作工具。

### 2.1 业务模式创新

| 维度 | 行业现状 | CMS-NG 的做法 |
|---|---|---|
| **AI 与人的关系** | 多为"AI 写、人工审"半自动 | **"AI 协作 + AI 自治"双轨**：人工协作是主线，全自动管道是补充 |
| **适用场景** | 多聚焦"单篇写作"工具 | **覆盖编辑部完整生产链路**：热点 → 选题 → 写稿 → 审稿 → 多平台分发 |
| **内容分发** | 多为导出 Markdown / 复制粘贴 | **原生 5 平台适配器**，每平台独立 prompt 与后处理 |
| **内容安全** | 主要靠人工把关 | **半成品保留 + 敏感词过滤 + 撤回机制 + 全局 Kill Switch** 四道兜底 |

**最大差异化**：市面上大多数 AI 内容工具定位"辅助个人创作者"，CMS-NG 直接定位为**"媒体机构编辑部操作系统"**——它管理的是角色、流程、流水线与发布，而不仅仅是单篇文章。

### 2.2 产品体验创新

#### 内容级语言原生支持

系统原生支持四种内容语言风格：**简体中文 / 港式繁体 / 粤语口语点缀 / 英文**。从记者偏好语言 → AI prompt 指令注入 → 多平台输出语言形成端到端可配置链路——同一篇稿件可以被要求同时产出"港式繁体书面语"和"粤语口语点缀"两个版本，且每种风格都有独立的 prompt 调优。

#### 内容本体与平台表达解耦

每个平台适配器独立维护"我的内容长什么样"——Instagram 强制首图 1:1，小红书自动加 emoji 与话题，Facebook 长度截断与 hashtag 风格化。这种"内容本体与平台表达解耦"的设计，让内容工程化成为可能：作者只需关心稿件本身，平台分发完全交给系统。

#### 失败隔离的管道执行

自动管道的每一步失败**只影响当前文章**，不会阻断同一批次其他文章；失败步骤与错误信息被持久化，便于**单篇级别重试**。这种"原子化 + 局部失败"模型让"全自动"不等于"不可控"——局部失败的成本从"重头再来"降为"接着改"。

### 2.3 部署与运维创新

- **外部中间件策略**：不与具体云厂商绑定，同一份部署配置既可跑在阿里云也可跑在 AWS 上；
- **地域感知的代理开关**：通过环境变量即可在"大陆开发（走代理）"与"海外生产（直连）"之间切换，**业务代码零修改**；
- **AI 模型切换零成本**：支持 DeepSeek、Kimi、OpenAI 等多个 AI 提供商无缝切换，所有 prompt、工具、审计、调用统计全部沿用。

### 3.2 效率价值

| 场景 | 传统人工 | CMS-NG 智能化 | 提升 |
|---|---|---|---|
| 找选题 | 编辑人工浏览 10+ 个网站，30–60 分钟 | AI 聚合 12 路热点 + 个性化推荐，**30 秒** | **60–120×** |
| 资料调研 | 1 篇深度稿需 2–4 小时人工搜索 | AI 联网检索，**1–2 分钟** | **60–120×** |

**典型场景实测**：以 1 篇 800 字科技快讯为例——
- 传统流程：选题 30min + 调研 60min + 写作 45min + 配图 15min + 排版 15min + 分发 5 平台 30min = **3 小时 15 分钟**；
- CMS-NG 协作流程：选题导入 1min + AI 调研 2min + AI 初稿 2min + 人工润色 10min + 1 键分发 30s = **约 15 分钟**；
- **单篇效率提升约 13 倍**。

| **事实核查** | AI 联网核查关键论断并标注置信度 | 编辑对 AI 出稿的信任度提升，审核时间下降 50%+ |
| **AI 审稿报告** | 多维度打分（事实 / 立场 / 敏感 / SEO / 读者友好度） | 编辑有"第二双眼睛"做交叉验证 |

### 3.5 智能化带来的"二阶价值"

除直接效率与成本外，CMS-NG 的智能化还会带来几个**长尾价值**：

1. **内容产量天花板被打破**：自动管道可在凌晨 / 节假日继续生产，对"日更热点"类账号尤其有价值；
2. **编辑能力被放大**：初级编辑在 AI 辅助下能产出原本需要高级编辑才能把控的稿件；
3. **数据资产沉淀**：所有 AI 调用、稿件状态、发布结果都进入数据库，长期可训练内部"选题 → 爆款"模型；
4. **风险可控**：半成品保留 + 撤回机制 + 全局 Kill Switch 三道阀门让"全自动"不等于"不可控"。

## 4. 总结

CMS-NG 的产品哲学可以概括为三句话：

1. **AI 是记者的"协作伙伴"而不是"替代者"**——14 项 AI 原子能力 + 9 状态稿件生命周期让协作发生在"最需要协作的环节"；
2. **AI 自治是"人力的补充"而不是"风险的来源"**——自动管道配套半成品保留、单篇重试、全局 Kill Switch、敏感词过滤四道兜底；
3. **产品定位为编辑部操作系统**——它管理的是角色、流程、流水线与发布，而不仅仅是单篇文章。

它不是市面上"又一个 AI 写作工具"，而是为媒体编辑部量身定制的**完整内容生产操作系统**。对于希望以最小人力成本覆盖多平台高频更新的媒体机构，CMS-NG 提供了一条**经过工程化验证**、**可平滑演进**、**风险可控**的智能化升级路径。
