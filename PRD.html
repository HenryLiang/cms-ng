01创作大脑

以AI智能体为核心的新一代内容创作作业系统
让每位记者都拥有一支AI特攻队

📄 产品需求文档 🏢 LC 传媒 📅 2025年5月 🔖 v1.1

1. 产品概述

1.1 产品背景

LC作为香港领先的新媒体平台，拥有超过200人的内容创作团队，每日产出大量新闻专题报道及社交媒体内容。当前团队使用自研传统CMS系统，内容创作流程仍高度依赖人工，在选题发现、资料搜集、多平台适配等环节存在效率瓶颈。

随着AI技术成熟，我们有机会以AI智能体（AI Agent）为核心，重构内容创作的全流程，将AI从"辅助工具"升级为"创作伙伴"，释放记者的创造力，提升内容生产的效率与质量。

1.2 产品定位

产品名称：01创作大脑（代号：CMS-NG）

产品定位：面向新闻内容团队的人机协作创作平台

核心价值主张：让每个记者都拥有一支7×24小时待命的AI特攻队，覆盖选题、研究、写作、编辑、审核、分发全流程，使记者专注于深度思考与价值判断，将重复性工作交给AI。

1.3 目标用户

2. 产品愿景

2.1 核心理念：「一个记者 + N个AI伙伴」

传统新闻生产中，一个记者需要独自完成选题、采访、写作的全过程。在01创作大脑中，每位记者入职时即配置一支专属AI特攻队，团队成员包括：

🎯

选题智能体

24小时监控热点，基于记者专长推送个性化选题，标注竞争度与潜在影响力。

📚

研究智能体

自动搜集网络资料、整理事件时间线、绘制人物关系图谱、标注信息来源可信度。

✍️

写作智能体

学习记者的写作风格，生成贴合个人习惯的初稿，支持多种风格切换与多语言改写。

🔍

编辑智能体

优化标题与结构、调整语气、检查逻辑漏洞，提供多个优化版本供选择。

⚖️

审核智能体

事实核查、法律风险评估、公关风险预警，重要稿件触发多智能体联合评审。

📢

分发智能体

自动适配多平台格式、生成配套素材、优化发布时间与策略。

2.2 人机协作原则

AI主导执行，人类主导决策。

AI不会自动发稿。所有AI生成的内容必须经过人类编辑审核、修改、冠名后才能发布。AI是"隐形助手"，人类是"台前作者"。

系统遵循以下协作原则：

  1. 透明性：AI的每一个建议都必须说明理由和依据，不可"黑箱操作"。
  2. 可控性：记者随时可以接管、修改或拒绝AI的建议，AI必须服从人类指令。
  3. 渐进性：AI从辅助简单任务开始，逐步深入到复杂创作环节，让记者逐步建立信任。
  4. 个性化：AI学习每位记者的风格和偏好，越用越"懂"记者。
  5. 可追溯：所有AI参与的操作都留下记录，方便回溯和责任界定。

4. 模块一：记者工作台 (Reporter Workspace)

MVP核心模块 第一优先

4.1 功能概述

记者工作台是每位记者每天打开系统后看到的第一个界面，也是与AI智能体互动最频繁的场所。设计目标是让记者在一个界面内完成从"看任务"到"交稿件"的全流程。

4.2 核心功能

4.2.1 我的看板 (My Kanban)

类似Trello的看板视图，展示记者当前参与的所有选题/稿件，按状态分为若干列：

  • 选题中 — 已创建选题，正在确定方向和搜集资料
  • 采写中 — 正在撰写初稿，可与AI协作写作
  • AI优化中 — 初稿已完成，AI正在进行标题优化、润色等
  • 编辑审核中 — 已提交编辑审核，等待反馈
  • 已发布 — 审核通过并已发布，可查看效果数据

每张卡片展示：选题标题、最后更新时间、AI协作进度指示器、下一行动提示（如"AI正在生成初稿，预计还需2分钟"）。

4.2.2 AI协作面板 (AI Copilot Panel)

常驻在编辑器右侧（或可收起为悬浮按钮）的AI对话面板。记者可以随时与AI对话，执行以下操作：

指令类型 示例 AI响应
续写/改写 "帮我把这段改得更口语化" 生成改写版本，标注修改处
标题生成 "给这篇稿子起5个标题" 生成5个标题+每个的预估效果
资料补充 "帮我找一下这个人的背景" 搜索并返回结构化资料
逻辑检查 "这段论述有没有漏洞？" 标出逻辑薄弱点+改进建议
风格切换 "改成严肃新闻风格" 全文或选中段落风格转换
摘要生成 "生成一段100字的摘要" 生成多长度版本摘要

4.2.3 智能编辑器 (Smart Editor)

基于富文本编辑器的创作界面，深度集成AI能力：

  • AI快捷菜单：选中任意文本，弹出悬浮菜单，一键执行"改写/润色/缩短/扩写/翻译"
  • 实时建议：AI在写作过程中实时提供建议（如"这里可以补充一个数据"），以非侵入式提示呈现
  • 结构化辅助：支持大纲模式，AI可基于选题自动生成文章大纲
  • 版本历史：自动保存所有修改版本，支持AI修改前后的对比查看
  • 协作光标：多人同时编辑时，显示其他人的光标位置和编辑内容

4.2.4 采访助手 (Interview Assistant)

针对采访环节的智能辅助工具：

  • 录音转录：上传采访录音，AI自动转录为文字，识别不同说话人
  • 关键信息提取：从转录文本中自动提取关键观点、金句、数据
  • 人物关系图谱：基于采访内容自动生成涉及人物的关系图
  • 采访提纲生成：基于选题背景，AI生成结构化采访提纲

5. 模块二：选题中心 (Story Hub)

MVP核心模块 第一优先

5.1 功能概述

选题中心解决"今天写什么"的核心问题。它不仅是热点聚合面板，更是AI驱动的选题发现与评估系统，帮助记者在信息洪流中快速锁定有价值的选题。

5.2 核心功能

5.2.1 热点雷达 (Trending Radar)

实时聚合多源热点数据的可视化面板。数据源按优先级分四级接入（详见附录C），确保LC 传媒记者掌握最全面、最及时的热点信息：

优先级 数据层 代表源 覆盖维度
P0 核心信源 Google Trends、新华网、路透、BBC、联合早报 香港本地热搜、涉港政策、全球财经、国际政治
P1 扩展信源 NewsAPI、澎湃新闻、卫报、RSSHub社交、36氪 全网聚合、深度调查、社交热点、科技商业
P2 补充信源 德国之声、环球网、中国政府网 欧洲视角、国际热点、权威政策原文
P3 评估中 微博热搜API、头条热榜、CNN 国内社交舆情、算法热点、美国视角
  • 热度指标：AI计算综合热度分数，显示趋势方向（上升/下降/平稳）
  • 竞争度分析：标注已有多少家媒体跟进报道，提示"独家机会"或"红海提醒"
  • 关联推荐：点击一个热点，AI推荐相关的延伸选题角度
  • 信源标签：每条热点标注来源类型（官方/国际/社交/聚合），方便记者判断可信度

5.2.2 AI选题推荐 (AI Story Suggestions)

每天为每位记者个性化推送3-5个选题建议：

  • 个性化匹配：基于记者的专长标签、历史选题、兴趣方向进行匹配
  • 推荐理由：每个推荐都附带AI的推荐理由（"这个选题与你的社会线专长匹配度高，且目前竞争度低"）
  • 难度预估：AI预估选题的采访难度、写作难度、潜在风险等级
  • 时效提醒：标注选题的"最佳报道窗口期"，提示紧迫性

5.2.3 一键立项 (One-Click Story Creation)

从选题到项目的无缝衔接：

  • 点击感兴趣的热点 → 弹出快速创建窗口
  • 输入选题标题和简要描述 → 系统自动创建稿件项目
  • AI自动启动：立项同时，研究智能体自动开始搜集背景资料
  • 记者在工作台"选题中"列立即看到新项目，附带AI已搜集的初步资料

5.2.4 选题碰撞 (Story Brainstorm)

与AI讨论选题可行性的对话空间：

  • 记者可以向AI提问："这个角度有没有新意？"、"还有哪些信息源可以利用？"
  • AI基于企业知识库和实时数据给出分析建议
  • 支持"多角度发散"：AI生成同一选题的5-10个不同报道角度

6. 模块三：AI协作创作 (AI Co-creation)

MVP核心模块 第一优先

6.1 功能概述

AI协作创作是系统的核心能力层，支撑记者在写作全流程中与AI智能体深度协作。它不仅是一个"写作工具"，更是一个"创作伙伴"，能够参与资料搜集、初稿生成、风格调整、多平台改写等各个环节。

6.2 核心功能

6.2.1 智能资料搜集 (Smart Research)

当记者创建选题后，AI自动启动资料搜集流程：

1

关键词提取

AI分析选题标题和描述，提取核心关键词和关联概念

↓
2

多源搜索

同时搜索新闻数据库、社交媒体、公开报告、历史报道等多源信息。自动从 Wikipedia 获取相关词条摘要（中英文），作为事实核查的基准来源

↓
3

信息整理

将搜集到的信息整理为结构化资料包：事件时间线、关键人物、核心数据、各方观点

↓
4

可信度标注

每条信息标注来源和可信度评级（官方确认/多方报道/单一来源/待核实）

最终生成的资料包包含以下内容：

Wikipedia 资料增强（v1.5 新增）

在生成资料包时，系统自动从 Wikipedia 搜索相关词条（中英文并行），将词条摘要注入 AI 作为事实依据。AI 基于这些真实资料进行整理，而非仅依赖模型内部知识。每条 Wikipedia 引用的信息均标注来源链接，记者可一键跳转原文核实。

触发方式：全自动，记者无感知。根据选题标题自动搜索，无需手动输入。
语言策略：优先搜索中文（zh / zh-yue），英文作为补充，结果自动合并去重。
降级策略：搜索无结果或 API 失败时，静默降级为纯 AI 生成，不影响使用体验。

资料包时效性增强（v1.7 已实现）

原问题：v1.5 发现 AI 生成的资料包中大量信息为一年多以前的内容，对时效性强的新闻选题参考价值有限。

解决方案：接入联网实时搜索能力，将最新资讯注入 AI 作为上下文，根本解决知识截止日期限制。

实时搜索增强实现细节（v1.7）

生成资料包时，系统在调用 AI 整理前，自动执行以下步骤：

  1. 联网搜索：根据选题标题自动触发联网搜索，获取最新新闻和动态
  2. 搜索结果注入：将搜索到的资讯摘要作为上下文注入 Prompt，与 Wikipedia 资料合并
  3. AI 整理：AI 基于实时搜索结果 + Wikipedia 资料生成结构化资料包，优先使用最新信息
  4. 静默降级：搜索失败时自动回退到纯 AI 生成 + Wikipedia，不影响使用体验

双搜索源架构

系统支持通过 SEARCH_PROVIDER 环境变量切换搜索提供商：

  • Kimi 内置搜索（默认）：使用 Kimi 模型的 $web_search 工具，无需额外 API Key
  • Tavily 搜索：专业 AI 搜索引擎，支持深度搜索模式、时间范围过滤、新闻主题优化。需配置 TAVILY_API_KEY

两者可无缝切换，生产环境建议根据实际效果选择。

6.2.2 初稿生成 (Draft Generation)

基于资料包和记者的写作指令，AI生成结构化初稿：

  • 风格选择：记者可选择"严肃新闻"、"深度分析"、"轻快报道"等预设风格，或让AI学习记者历史稿件的风格
  • 大纲确认：AI先生成大纲 → 记者确认或修改 → 再生成全文，避免方向偏差
  • 引用标注：初稿中所有引用事实都自动标注来源链接，方便记者核实
  • 多角度版本：同一选题可生成侧重不同角度的多个初稿版本供选择

重要设计原则：AI生成的初稿必须在编辑器中以"建议模式"呈现（类似Word的修订模式），所有AI生成的内容都用特殊颜色/背景标注。记者可以选择"接受"、"修改"或"拒绝"每一处AI建议。AI不会直接覆盖记者的原创内容。

6.2.3 标题实验室 (Headline Lab)

专业的标题优化工具：

  • AI生成：基于全文内容生成5-10个标题选项
  • 效果预测：每个标题附带AI预测效果（点击率预估、情感倾向、搜索引擎友好度）
  • A/B测试：支持创建多个标题版本，发布后追踪实际表现
  • 风格切换：同一内容生成"严肃版"、"悬念版"、"数字版"等不同风格标题

6.2.5 事实核查助手 (Fact-check Assistant)

在写作和审核阶段自动进行事实核查：

  • 自动标注：AI扫描全文，自动标注所有涉及的事实性陈述（人名、地名、时间、数据）
  • 来源建议：对每一处事实，AI建议最可靠的核实来源
  • 一致性检查：检查全文内部逻辑一致性（如前面说"10人"后面说"12人"）
  • 争议标注：对有争议或多方不同说法的信息，AI标注"多方说法不一，建议补充来源"

7.2.1 AI预审报告 (AI Pre-review Report)

7.2.2 多智能体联合评审 (Multi-Agent Review)

对于敏感或重要稿件（涉及政治、法律、企业舆情、公众人物等），系统自动触发多智能体联合评审机制。评审由4个独立AI智能体并行执行，每个智能体专注于不同维度：

智能体 评审维度 输出格式

触发条件：

  • AI预审评分低于B级（即存在多项事实或法律风险）
  • 文章标签包含「政治」「法律」「企业」「公众人物」等敏感分类
  • 编辑手动触发「请求联合评审」

评审结果呈现：

  • 顶部汇总：4个智能体的综合风险等级（绿色/黄色/红色），编辑一键决策
  • 分栏展示：每个智能体的评审意见独立展示，支持按风险等级筛选
  • 原文批注：直接在文章对应段落旁显示相关智能体的标注（类似Google Docs评论）
  • AI辩论：当两个智能体意见冲突时（如法律智能体认为安全但公关智能体认为高风险），系统高亮显示冲突点，提示编辑特别关注

编辑决策：编辑可「采纳全部建议」「逐条确认」「驳回」或「转交法务/公关部门」。所有决策记录到审核日志。

7.2.3 看板式审核 (Kanban Review)

编辑的工作看板，展示所有待审核稿件：

  • 按紧急程度排序（突发新闻优先）
  • AI预审评分帮助编辑判断审核优先级
  • 支持快速操作：通过 / 退回修改 / 拒绝
  • 退回修改时，AI可辅助生成修改建议

7.2.4 版本对比与批注 (Version Compare & Annotation)

编辑在审核过程中需要对比稿件的多个版本，理解记者的修改意图，并留下专业的审核意见。

版本对比（Diff View）：

  • 三栏对比模式：原始稿 | AI优化版 | 记者修改版，并排展示，差异以色块高亮（新增=绿色、删除=红色、修改=黄色）
  • 变更导航：侧边栏显示所有变更点的缩略列表，点击快速跳转
  • 逐段对比：点击任意段落展开该段的全部版本历史，包括修改人、修改时间、修改原因（记者提交时可填写）
  • 忽略格式差异：支持只对比内容变化，忽略标点、空格的微调

批注系统（Annotation）：

  • 行内批注：选中任意文本段落，弹出批注浮层，输入审核意见。批注以彩色竖条标记在文本左侧
  • 批注类型：提问（❓）、修改建议（✏️）、事实核查请求（🔍）、风险提示（⚠️）、通过标记（✅）
  • @AI 辅助：在批注框输入「@AI 帮我改写这段」「@AI 核实这个数据」，AI自动响应并生成建议
  • 批注回复：记者收到退回稿件后，可在每条批注下回复说明修改情况，形成审核对话线程
  • 批注状态：未读 → 已读 → 已处理 → 已解决，编辑可逐条确认关闭

审核历史追溯：

  • 时间轴视图：按时间顺序展示该稿件的全部审核事件（提交→AI预审→编辑批注→退回→修改→再审→通过）
  • 操作人记录：每个操作（修改、批注、通过、退回）记录操作人、时间、IP地址
  • 导出审核报告：一键导出PDF格式的审核报告，包含全部批注、修改对比、最终版本

8. 模块五：多平台分发中心 (Distribution Hub)

第二优先

8.1 功能概述

多平台分发中心是AI驱动的"内容适配工厂"，自动将深度报道转化为适合不同平台的格式和风格。核心模式为"AI改写 → 平台预览 → 保存版本 → 一键复制 → 人工粘贴"。

关于自动发布

第三方平台（小红书、微信、Facebook、Instagram、微博等）均不提供稳定可靠的官方发布API，或API审核门槛极高、权限受限。系统不做API自动发布，采用"AI辅助生产，人工完成最后一公里"策略。此策略既保证了内容质量与合规性，又避免了API对接的复杂性和账号风险。

8.2 核心功能

8.2.1 AI平台改写 (AI Platform Adaptation)

审核通过后，为每个目标平台生成适配版本：

  • 一键生成：选择目标平台，AI自动改写标题、正文、标签
  • 平台风格库：每个平台有预设的改写风格（种草风、短平快、长文深度等）
  • 封面图提示词：AI生成适配该平台风格的封面图提示词，供设计师参考
  • 多版本管理：同一稿件可保存多个平台的适配版本，独立管理

8.2.2 平台适配预览 (Platform Preview)

在发布前，模拟各平台的呈现效果，让运营人员直观了解内容在目标平台的最终展示形态，避免发布后才发现格式问题。

支持平台与预览模式：

平台 预览模式 展示元素
小红书 手机端笔记卡片 封面图（3:4）+ 标题（最多20字）+ 正文前3行折叠 + 底部标签区
微信公众号 公众号文章页 封面大图 + 标题 + 作者信息 + 导语 + 正文（支持图文混排预览）
微博 微博信息流卡片 用户头像+昵称 + 正文（最多2000字，超140字显示「展开」）+ 配图九宫格 + 底部互动栏
Facebook Facebook Feed帖文 发布者信息 + 正文 + 链接卡片（OG图+标题+描述）+ 互动按钮
Instagram Instagram Feed / Stories Feed：正方形图片/轮播 + 短文案（首行显示，点击展开）+ 标签区
Stories：竖屏9:16图片/视频 + 顶部文字叠加
X (Twitter) X帖文卡片 用户头像+昵称 + 正文（280字限制，超出截断）+ 配图（最多4张2x2网格）
LinkedIn LinkedIn文章/动态 发布者职业信息 + 正文 + 链接预览卡片（1.91:1横图）+ 专业互动栏

预览交互：

  • 设备切换：支持在「桌面端」和「移动端」之间切换，模拟不同设备的显示效果
  • 深色模式：支持切换浅色/深色模式预览（部分平台支持）
  • 实时编辑：直接在预览中修改标题和正文，修改实时同步到对应平台的适配版本
  • 字数超限提示：当正文超过平台限制（如X的280字、小红书标题20字）时，预览区红字提示并在对应位置截断显示
  • OG图预览：在Facebook/LinkedIn预览中，显示AI生成的OG图或占位图，点击可唤起「AI智能配图」面板生成/替换

分屏对比：运营可同时打开2个平台的预览窗口并排对比（如小红书 vs Instagram），确保内容在不同平台呈现一致的核心信息。

8.2.3 一键复制 (One-Click Copy)

运营完成平台适配和预览确认后，需要快速将内容复制到目标平台的发布界面。系统提供多种复制方式，覆盖不同平台的内容格式需求。

复制模式：

复制类型 复制内容 适用场景
完整文案 标题 + 正文 + 标签/话题（按平台格式自动拼接） 直接粘贴到平台发布框
仅标题 适配后的平台标题 平台标题与正文分开填写时（如微信公众号）
仅正文 适配后的正文（保留段落格式，自动转换平台支持的富文本标记） 正文需要单独粘贴到编辑器
仅标签 #话题1 #话题2（按平台格式，X用#，Instagram用#，小红书用#） 单独调整标签时使用
封面图提示词 AI生成的封面图prompt + 推荐尺寸 交给设计师制作封面图

复制格式智能转换：

  • 小红书：正文自动添加段落间的emoji分隔符，标签自动添加#前缀，标题限制20字超出截断并提示
  • 微信公众号：正文保留Markdown格式（标题/加粗/引用/链接），发布前可一键转为微信编辑器兼容的HTML
  • 微博：正文超过140字时自动提示「长文」标识，图片链接自动转为微博配图上传提示
  • Facebook/LinkedIn：正文保留段落格式，链接自动转为OG链接卡片格式
  • X：正文严格限制280字符（含标点），超出时红色高亮并禁止复制，提示精简
  • Instagram：正文首行提取为短文案预览，剩余内容标记为「展开」，标签集中放在末尾

批量导出：

  • 多平台文案包：一次性导出所有已适配平台的文案（Markdown格式，按平台分章节）
  • 纯文本包：导出为TXT格式，无格式标记，适合直接粘贴
  • JSON数据包：导出结构化JSON（标题/正文/标签/封面图URL/发布状态），供第三方工具或自动化脚本使用

发布状态标记：

  • 运营粘贴发布后，在对应平台卡片点击「标记为已发布」，系统记录发布时间
  • 支持「已复制（待发布）」「已发布」「发布失败（需重试）」三种状态
  • 全部平台发布完成后，系统自动更新稿件状态为「已分发」

8.2.4 效果追踪 (Performance Tracking)

内容分发到各平台后，需要持续追踪传播效果，为后续内容策略优化提供数据支撑。效果追踪采用「运营手动回填核心数据 + AI自动分析趋势」的人机协作模式。

核心数据指标：

指标类别 指标名称 数据来源 录入方式
曝光类 阅读量 / 浏览量 / 展示次数 各平台后台 运营手动回填（每日/每周）
互动类 点赞数、评论数、转发/分享数、收藏数 各平台后台 运营手动回填
粉丝类 新增粉丝、净增关注 各平台后台 运营手动回填
转化类 点击率（CTR）、跳转官网次数 官网 Analytics + 平台后台 系统自动采集（UTM参数追踪）
负面类 举报数、负面评论占比 各平台后台 运营手动回填

数据回填 UI：

  • 分发任务卡片：每篇稿件的分发面板显示各平台的「待回填」状态，运营点击后弹出录入浮层
  • 批量录入：支持Excel/CSV导入，格式为「文章ID | 平台 | 日期 | 阅读量 | 点赞 | 评论 | 转发 | 收藏」
  • 数据校验：录入时自动校验数值合理性（如阅读量必须 ≥ 点赞数），异常值黄色高亮提示
  • 历史趋势：每个指标显示近7天/30天趋势折线图，帮助运营判断数据波动

AI 智能分析：

  • 高表现内容特征提取：AI自动分析高阅读/高互动内容的共性（标题特征、发布时间、话题类型、配图风格、正文长度等），输出《高表现内容特征报告》
  • 平台适配度评分：同一内容在不同平台的表现差异分析，AI给出「该内容更适合哪个平台」的建议
  • 异常预警：当某平台数据突然下滑（如阅读量环比下降50%）或某篇内容互动率异常低时，AI自动推送预警通知
  • 竞品对比：录入竞品账号的公开数据后，AI生成竞品内容策略对比报告

趋势报告自动生成：

  • 周报告（每周一自动生成）：本周总发稿量、各平台分发量、Top 3 高表现文章、Top 3 低表现文章、AI优化建议
  • 月报告（每月1日自动生成）：月度传播数据汇总、平台表现排名、内容类型分布、粉丝增长趋势、下月选题建议
  • 专题报告（手动触发）：针对特定选题或活动（如「香港书展报道」）生成专项传播效果分析
  • 报告导出：支持导出为PDF（带图表）或PPT格式，直接用于团队会议

效果数据与内容创作联动：

  • 在「选题中心」展示「历史同类选题表现」，帮助记者判断选题潜力
  • 在「AI协作面板」中，当记者要求AI生成标题时，AI参考高表现历史内容的标题特征进行优化
  • 在「编辑审核台」中，编辑可查看该记者过往稿件的平均传播表现，作为质量评估参考

9. 模块六：管理后台 (Command Center)

第三优先

9.1 功能概述

管理后台面向主编和管理层，提供全局视角的内容生产管理和数据分析能力。

9.2 核心功能

9.2.1 团队工作看板 (Team Dashboard)

为主编和管理层提供全局视角的内容生产实况，支持实时决策和资源调配。

看板布局（三栏式）：

  • 顶部 KPI 横幅：今日发稿量 / 本周发稿量 / 待审核稿件数 / 平均审核耗时 — 实时刷新，红色预警当审核队列 > 10 篇或平均审核耗时 > 4 小时
  • 左栏：生产流水线：按稿件状态分泳道（选题中 → 采写中 → 待审核 → 审核中 → 待发布 → 已发布），每道显示稿件卡片（标题/记者/截止时间），支持拖拽调整优先级
  • 中栏：记者负载：每位记者的当前任务数、本周已完成数、平均稿件质量评分、当前状态（空闲/正常/过载）。过载（任务 > 5）时红色高亮
  • 右栏：热点覆盖：当前热点雷达中的 TOP 10 话题，每个话题显示「已覆盖/未覆盖」状态、已发布稿件数、阅读量汇总

预警与通知：

  • 审核积压预警：审核队列超过 10 篇或单篇超过 4 小时未审，自动推送给主编和对应编辑
  • 截稿风险预警：距离截稿时间 < 2 小时且状态未达「待审核」的稿件，黄色高亮并推送记者
  • 热点遗漏预警：TOP 5 热点中超过 2 个未覆盖时，推送选题主编建议安排报道
  • 记者过载预警：某记者同时负责选题 > 5 个时，提示主编考虑任务再分配

数据钻取：点击任意数字/卡片可下钻查看详情。如点击「待审核 8 篇」展开审核队列列表；点击记者姓名查看该记者的全部进行中和历史稿件。

9.2.2 AI使用分析 (AI Analytics)

全面追踪AI功能在内容生产中的实际价值，为功能迭代和推广策略提供数据支撑。

核心指标仪表盘：

指标维度 具体指标 计算方式
功能使用率 各AI功能的月活跃使用次数、使用记者数、人均使用频次 从AIOperation表按agentType聚合
功能渗透率 使用过AI功能的记者占比、稿件中至少使用过一项AI功能的占比 活跃记者数 / 总记者数
效率提升 AI辅助稿件的平均创作时长 vs 纯人工稿件的平均创作时长 （纯人工时长 - AI辅助时长）/ 纯人工时长 × 100%
质量对比 AI参与稿件的审核通过率、编辑评分均值 vs 纯人工稿件 对比两组稿件的审核评分均值
AI采纳率 AI生成建议中被记者/编辑实际采纳的比例（如AI标题被采用率） 采纳次数 / 生成次数 × 100%
成本效率 AI API调用成本 / 节省的人工工时，计算ROI （节省工时 × 人力成本）/ API成本

记者AI画像分层：

  • AI重度用户（周使用 > 20次）：高频使用多个AI功能，通常效率提升显著。作为内部推广的种子用户，邀请撰写使用心得。
  • AI中度用户（周使用 5-20次）：主要使用1-2个核心功能（如续写、标题生成）。通过推送新功能教程和成功案例，引导扩展使用场景。
  • AI轻度用户（周使用 < 5次）：偶尔尝试，对AI能力持观望态度。安排1对1培训或推荐"最快上手的3个AI功能"。
  • AI未激活用户（注册后从未使用）：分析阻碍原因（是不了解功能入口？还是尝试后不满意？），针对性推送激活引导。

AI效果归因分析：

  • 单篇归因：每篇稿件详细记录使用了哪些AI功能、使用次数、AI输出是否被采纳。分析「AI标题生成+采纳」vs「纯人工标题」的传播效果差异。
  • 功能价值排序：按「采纳率 × 使用频次」综合评估各AI功能的价值贡献，识别低价值功能（使用少+采纳低）进行优化或下线。
  • 时段分析：分析AI使用高峰时段（通常是截稿前），评估是否需要在高峰时段增加AI服务资源保障响应速度。

可视化报表：

  • 周度简报（自动发送至主编邮箱）：本周AI使用总次数、Top 3 活跃功能、效率提升数据、待优化功能提示。
  • 月度深度报告：AI功能使用趋势图（近6个月）、记者分层饼图、质量对比柱状图、ROI计算。
  • 自定义报表：支持按时间段、按部门、按记者筛选生成定制报表。

9.2.3 知识库管理 (Knowledge Base Management)

知识库是01创作大脑的智能资产中心，为AI功能提供领域知识支撑，同时为记者和编辑提供高效的资料检索能力。

知识类型与结构：

知识类型 内容示例 维护方式 AI用途
历史报道库 已发布的全部文章，含标题、正文、标签、发布时间、阅读量 自动归档（发布后自动入库） AI续写时参考同类报道；资料搜索时语义检索
人物档案 常报道人物的基本信息、职务变动、关联事件、立场倾向、采访历史 编辑手动维护 + AI自动提取文章人物信息辅助生成 AI写作时自动引用人物背景；记者快速了解报道对象
机构档案 政府部门、企业、NGO等机构的背景、近期动态、发言人信息 编辑手动维护 AI在涉及相关机构时自动补充背景说明
风格指南 编辑规范、标题撰写规范、敏感词库、禁用表述、标点使用规则 主编维护，版本化管理 AI润色和风格检查的依据
热点事件追踪 持续追踪的重大事件的时间线、关键节点、各方表态 编辑手动维护 + AI自动提取文章时间线辅助更新 AI续写时提供事件上下文；记者快速跟进进展
优质稿件样本 编辑评选的高分稿件（审核评分A+或传播表现Top 10%） 编辑标记为「优质样本」后自动入库 AI few-shot学习的样本，提升生成质量

检索能力：

  • 全文检索：基于关键词的传统搜索，支持布尔逻辑（AND/OR/NOT）和短语精确匹配
  • 语义检索：基于向量嵌入的语义搜索，输入自然语言描述即可找到相关内容（如搜索「最近关于楼市降温的报道」）
  • 多条件筛选：按时间范围、作者、标签、稿件状态、阅读量级等组合筛选
  • 关联推荐：查看某篇稿件时，自动推荐「相关报道」「同一人物的其他报道」「同一事件的后续报道」

AI 辅助维护：

  • 自动标签提取：新文章发布时，AI自动提取关键词和主题标签，建议入库标签，编辑确认后生效
  • 人物信息自动更新：当新文章中出现已建档人物的新职务或新事件时，AI建议更新人物档案，编辑审核后合并
  • 重复检测：新文章入库时，AI自动检测与历史报道的重复度，高重复时提示编辑判断是否需合并或更新
  • 知识库健康度检查：定期扫描知识库，标识「过期信息」（如人物职务已变动但档案未更新）、「孤立条目」（无关联内容的孤立档案）

权限与审计：

  • 编辑权限：记者可检索和引用知识库内容，编辑可新增/修改人物和机构档案，主编可修改风格指南
  • 变更历史：知识库条目的每次修改记录版本差异和修改人，支持回滚到历史版本
  • 敏感信息保护：人物档案中的隐私信息（如联系方式、住址）仅对管理员和指定编辑可见，普通记者检索时自动脱敏

9.2.4 系统配置 (System Configuration)

  • AI模型配置：调整AI的创意程度、审核严格度等参数
  • 工作流配置：自定义审核流程、必填字段等
  • 平台接入：配置Facebook、Instagram等平台的API连接
  • 权限管理：角色与权限配置（记者/编辑/管理员）

14. 界面设计原则

14.1 设计系统

🎨

视觉风格

现代简洁的新闻编辑风格，以白色/浅灰为底，蓝色为主色，信息层级清晰。避免过度装饰，突出内容本身。

📐

布局原则

三栏自适应布局：左侧导航、中间内容、右侧AI面板。支持全屏编辑模式，AI面板可收起。

⚡

交互原则

减少点击层级，核心功能2步可达。AI响应时间<3秒，长时间操作显示进度和预计完成时间。

♿

无障碍设计

符合WCAG 2.1 AA标准，支持键盘导航、屏幕阅读器。高对比度模式可选。

14.2 AI元素视觉规范

为了建立用户对AI的信任感和辨识度，AI相关的视觉元素需要统一规范：

  • AI头像/图标：统一的机器人头像，不同智能体用不同颜色区分（选题=蓝色、写作=绿色、审核=橙色）
  • AI内容标识：AI生成的内容使用淡紫色背景 + "AI生成"标签，与人类内容明确区分
  • 思考状态：AI处理中显示动画指示器 + "正在思考..."文案 + 预计完成时间
  • 建议卡片：AI建议以卡片形式呈现，包含"接受 / 修改 / 拒绝"三个操作按钮

14.3 响应式设计

系统主要面向桌面端使用（记者和编辑的核心工作场景），但需支持：

  • 平板端：适配iPad等平板设备，支持触摸操作
  • 移动端（轻量版）：仅支持查看通知、快速审核、查看数据看板等轻量操作

  • 卡片包含：建议内容 + 理由说明 + 操作按钮
  • 点击"查看详情"可展开AI的完整分析
  • 支持"稍后再说"，将建议保存到待处理列表

15.3 AI错误处理

AI并非万能，系统需要优雅处理AI出错的情况：

场景 系统响应 用户操作
AI生成内容质量不佳 显示"内容质量较低，建议人工审核"警告 记者可以选择"重新生成"或"手动编辑"
AI无法完成请求 明确告知"暂时无法处理这个请求"，并说明原因 简化请求或尝试其他方式
AI生成可能不准确的信息 自动添加"⚠️ AI生成，请核实"标注 记者必须核实后手动移除标注
AI服务暂时不可用 显示离线状态，所有AI功能转为手动模式 可正常使用编辑器，AI功能稍后恢复

16.3 AI模型抽象层

为了未来能够灵活切换AI模型（如从Kimi切换到其他模型），系统设计统一的AI模型抽象层：

模型抽象层设计

不同的模型提供商（Kimi、OpenAI、本地模型）实现同一接口，上层业务代码无感知切换。

20. 术语表

术语 定义
AI智能体 (AI Agent) 具备特定专业能力、可自主执行任务并与人类协作的AI程序。在本系统中，每个智能体负责一个特定环节（如选题、写作、审核）。
选题 (Story) 内容创作的起点，指一个有待深入报道的新闻话题或事件。在系统中以项目形式管理。
稿件 (Article) 记者创作的完整内容作品，包含标题、正文、图片等，是系统的核心内容单元。
资料包 (Research Kit) AI为某个选题自动搜集并整理的结构化资料集合，包含时间线、人物、数据、观点等。
预审报告 (Pre-review Report) 稿件提交审核时，AI自动生成的质量评估报告，包含事实核查结果、风险提示和优化建议。
建议模式 (Suggestion Mode) 编辑器中的一种显示模式，AI生成的内容以特殊样式标注，记者可以选择接受、修改或拒绝每一处建议。
MVP Minimum Viable Product，最小可行产品。指以最小成本验证核心假设的第一个可用版本。
OG图 (Open Graph Image) 社交媒体上分享链接时显示的代表性图片。不同平台对OG图尺寸有不同要求（如Facebook 1.91:1、Instagram 1:1）。
Infographic 信息图表，将复杂数据或信息以可视化方式呈现的单张图片，常用于社交媒体传播和文章配图。
文变片 / 片变文 文变片指将文章转化为视频讲稿/字幕；片变文指将YouTube视频或音频转录改写为文章。实现跨媒介内容复用。
智能体生态 (Agent Ecosystem) 面向特定垂直领域（如股市、兽医、风水）的个性化AI对话机器人集合，可独立运营或面向商业客户。

C.7 技术接入规范

C.7.1 统一数据格式

所有数据源接入后需标准化为统一JSON结构：

{ "source": "xinhua-rss", "sourceName": "新华网", "sourceType": "official", // official / international / social / aggregate "title": "...", "description": "...", "url": "...", "publishedAt": "2025-05-17T08:00:00Z", "heatScore": 85, // 统一热度分 0-100 "trend": "rising", // rising / falling / stable "tags": ["政治", "香港"], "language": "zh-CN", // zh-CN / zh-TW / en "region": "mainland", // hk / mainland / international / global "rawData": { ... } // 原始数据保留 }

C.7.2 推荐接入架构

1

RSS 源直接拉取

新华网、路透、BBC、联合早报、人民网等公开RSS → rss-parser 库定时拉取

↓
2

RSSHub 兜底聚合

知乎热榜、微博热搜、澎湃新闻等无官方RSS → RSSHub 公共实例或自建 Docker 生成标准RSS

↓
3

API 聚合层

NewsAPI.org、卫报API、Google Trends → 统一封装为内部服务，按配额调度

↓
4

AI 去重与热度计算

不同来源的同一事件去重 → AI计算综合热度分 → 按记者专长标签匹配推荐
