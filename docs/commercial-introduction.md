## 一、商业价值呈现

### 1.1 它解决的是一个结构性矛盾

新媒体机构今天被困在一个越来越尖锐的矛盾里：**内容分发的平台越来越多，而能写、能审、能发布的人力却没有同步增长**。

一篇稿件从"看到热点"走到"出现在 5 个平台上"，传统流程要选题、调研、写作、配图、排版、逐平台搬运——单篇 800 字快讯耗掉一名记者 3 个多小时。与此同时：

- **凌晨的热点没人跟**，节假日更新断档；
- **初级编辑把控不住深度稿**，高级编辑成了瓶颈；
- **AI 生成的内容没人敢信也没人能审**，引入了新的质量风险；
- **多平台账号运维成本线性增长**，每多一个平台就多一份人力。

这些不是单点工具能解决的问题——它们是**生产关系层面**的问题。CMS-NG 的价值正在于此：它不是替记者"写得快一点"，而是**重塑整个编辑部的内容生产作业方式**，把人力从重复搬运中解放出来，集中到真正需要判断力的环节。

### 1.2 效率价值：单篇提升 13 倍

| 场景 | 传统人工 | CMS-NG | 提升 |
|---|---|---|---|
| 找选题 | 浏览 10+ 网站，30–60 分钟 | AI 聚合 12 路热点 + 个性化推荐，**30 秒** | **60–120×** |
| 资料调研 | 2–4 小时人工搜索 | AI 联网检索，**1–2 分钟** | **60–120×** |
| 写初稿 | 30–90 分钟 | AI 初稿生成，**1–3 分钟** | **10–30×** |
| 多平台分发 | 5× 复制粘贴 + 手工调格式 | **一键多平台，30 秒** | **30–50×** |
| 自动发布管道 | 编辑每日手动操作 | **完全无人值守，24×7** | ∞ |

**典型场景实测**——以 1 篇 800 字科技快讯为例：

- 传统流程：选题 30min + 调研 60min + 写作 45min + 配图 15min + 排版 15min + 分发 5 平台 30min = **3 小时 15 分钟**
- CMS-NG 协作流程：选题导入 1min + AI 调研 2min + AI 初稿 2min + 人工润色 10min + 一键分发 30s = **约 15 分钟**
- **单篇效率提升约 13 倍**

### 1.3 成本价值：人力 0 增量，产能 3 倍

| 成本项 | 传统模式 | CMS-NG 模式 | 节省 |
|---|---|---|---|
| 人力配置 | 5 人编辑部（3 记者 + 2 编辑）覆盖日更 5 平台 | 同等人力可覆盖日更 15+ 平台 | **人力 0 增量，产能 3×** |
| 配图成本 | 图库会员年费 5,000–20,000 元 | AI 配图按次计费，**约 0.1 元/张** | **>95%** |
| 调研工具 | 多个付费数据库 + 人工搜索时间 | 联网搜索 API（按量）+ AI 整合 | **>70%** |
| 多平台运维 | 多人多平台账号运维 | 集中配置 + 自动重试 | **运维 0 增量** |
| 标准化内容产能 | 0 篇（人力做不过来） | **每日 N 篇（可配置）** | **从 0 到 1** |

更关键的是**成本结构的变化**：传统模式下成本随平台数线性增长（多一个平台多一份人力）；CMS-NG 把"分发到第 N 个平台"的边际成本压到接近零——**规模越大，单位成本越低**，这是真正的规模效应。

### 1.4 质量与体验价值：不是"更快"，而是"更好且可控"

| 体验维度 | 实现机制 | 价值 |
|---|---|---|
| **个性化写作风格** | 记者设置擅长领域 + 偏好语言，AI 自动按风格出稿 | 一人多面（深度/快讯/评论）无需切换账号 |
| **多语种原生支持** | 内容级语言贯穿数据库 → prompt → 平台 | 简体/港式繁体/英文/粤语 4 种风格独立配置 |
| **事实核查** | AI 联网核查关键论断并标注置信度 | 审核时间下降 50%+，编辑对 AI 出稿信任度提升 |
| **AI 审稿报告** | 多维度打分（事实/立场/敏感/SEO/读者友好度） | 编辑有"第二双眼睛"做交叉验证 |
| **半成品保留** | 自动管道失败不丢稿 | 失败成本从"重头再来"降为"接着改" |
| **全程可审计** | 所有 AI 调用留痕（prompt/响应/token/耗时） | 满足媒体合规审计要求 |

### 1.5 战略价值：四个"二阶收益"

除了可直接量化的效率与成本，CMS-NG 还带来四个长尾战略价值——这些才是真正决定机构长期竞争力的部分：

1. **内容产量天花板被打破**——自动管道可在凌晨、节假日继续生产，对"日更热点"类账号尤其有价值。过去"做不过来就不做"的内容赛道，现在可以系统性覆盖。
2. **编辑能力被放大**——初级编辑在 AI 辅助下能产出原本需要高级编辑才能把控的稿件，**人才梯队建设的压力显著下降**。
3. **数据资产沉淀**——所有 AI 调用、稿件状态、发布结果都进入数据库，长期可训练机构专属的"选题 → 爆款"模型。**用的越久，数据壁垒越深**。
4. **风险全程可控**——半成品保留 + 撤回机制 + 全局 Kill Switch + 敏感词过滤四道阀门，让"全自动"不等于"不可控"。在媒体行业，**可控性就是生命线**。

### 1.6 一句话价值总结

> CMS-NG 让媒体机构用**相同的人力**覆盖**3 倍的平台**、产出**13 倍的速度**、维持**可审计的质量底线**，并在凌晨和节假日**继续生产**——这不是工具升级，而是生产方式的代际跨越。

## 二、产品功能描述

CMS-NG 不是"AI 写作工具"，而是一套覆盖编辑部完整生产链路的**全栈式内容生产操作系统**。下面按功能主线展开，重点说明**每个功能的创新之处**。

### 2.1 核心功能创新："AI 协作 + AI 自治"双轨模式

这是本产品区别于市面上所有同类工具的**根本性创新**。

| 维度 | 行业现状 | CMS-NG 的做法 |
|---|---|---|
| AI 与人的关系 | 多为"AI 写、人工审"的半自动 | **人工协作是主线，全自动管道是补充** |
| 适用场景 | 聚焦"单篇写作"工具 | **覆盖编辑部完整生产链路** |
| 内容分发 | 导出 Markdown / 复制粘贴 | **原生 5 平台适配器**，每平台独立 prompt |
| 内容安全 | 主要靠人工把关 | **四道兜底机制** |

市面上大多数 AI 内容工具定位"辅助个人创作者"，CMS-NG 直接定位为**媒体编辑部操作系统**——它管理的是角色、流程、流水线与发布，而不仅仅是单篇文章。**双轨模式**意味着：需要人工判断的深度内容走协作轨，标准化内容（快讯、数据报道）走自治轨，两条轨共享同一套 AI 能力、同一套数据、同一套审计——人力被精准投放到价值最高的环节。

### 2.2 全自动内容管道（核心差异化功能）

业内少见的**完全无需人工干预**的内容生产流水线，8 个阶段自动执行：

> 调度触发 → 选题采集（敏感词过滤 + 24h 去重）→ 资料调研（联网搜索）→ 文章生成 → 封面配图 → 保存文稿 → 平台发布 → 邮件汇报

**功能创新点不在"自动"本身，而在"自动却可控"的四道兜底设计**：

- **半成品保存**：失败步骤的中间产物落库，不丢失任何已完成工作——把"失败成本"从重做降到续做；
- **单篇重试 / 撤回**：失败文章可单独重试或从平台撤回，不影响整批；
- **全局 Kill Switch**：一个操作即可停止所有自动发布任务，防失控兜底；
- **敏感词过滤 + 邮件告警**：选题阶段就过滤高风险领域，任务结束立即汇总通知。

这意味着机构可以放心地把"标准化内容"交给管道在凌晨生产，把人力集中到"需要判断力的内容"上——**这是真正的产能杠杆**。

### 2.3 多平台分发：内容本体与平台表达解耦

已落地 **5 个真实可用适配器**，每个拥有**独立的 prompt 模板与后处理流水线**：

| 平台 | 适配点 |
|---|---|
| **Website**（自有站） | 站内发布、栏目分类、SEO 字段映射 |
| **Facebook** | 长度截断、Hashtag 风格化、配图比例 |
| **Instagram** | 首图强制 1:1、文案 emoji 化、9–30 个 hashtag |
| **小红书** | 标题口语化、emoji、3–5 个话题标签 |
| **WordPress** | 完整 REST API 集成（分类/标签/特色图） |

**功能创新点**：同一篇稿件在不同平台呈现完全不同的标题、长度与语气——不是简单截断，而是**按平台调性重新表达**。后端为每平台保存一份独立版本（`PlatformPublish` 表），避免平台间相互污染。**作者只需关心稿件本身，平台分发完全交给系统**——这是内容工程化的关键一步，也是"一稿分发 5 平台只需 30 秒"的底层支撑。

### 2.4 AI 协作能力矩阵：14 项原子操作

覆盖"写读审辅"四大类，全部以统一接口挂在稿件维度：

| 类别 | 能力 | 业务价值 |
|---|---|---|
| **写** | 改写 / 扩写 / 缩写 / 润色 / 多标题 / 摘要 / 对话写作 / 初稿生成 | 从一句话到完整稿件的全流程写作 |
| **读** | 资料调研（联网搜索）/ 事实核查（联网佐证）/ AI 选题推荐 | 替代大量人工搜索与调研 |
| **审** | AI 审稿（多维评分） | 辅助编辑交叉验证 |
| **辅** | SEO 优化 / AI 配图 | 提升可发现性与视觉吸引力 |

**两条不可妥协的原则**（这是功能设计上的创新，也是媒体行业的底线）：
- 所有 AI 输出**默认需人工确认**才能进入发布环节；
- 所有 AI 操作**全程留痕**（prompt、模型、token、耗时、结果），形成可审计轨迹。

### 2.5 记者工作台与编辑工作台

**记者工作台**让"找选题到交稿"顺滑无阻：选题中心（12 路 RSS 含 Google Trends/BBC/纽约时报/联合早报/36 氪/虎嗅 + X/Twitter 热点数据源）、9 状态稿件全生命周期管理、版本快照与任意历史版本回滚、AI 配图（Seedream）、个人专长与偏好自动加权。

**编辑工作台**让"守门人"有第二双眼睛：统一审稿队列、分配编辑与退回修改、AI 审稿报告（事实/立场/敏感词/SEO/读者友好度多维评分）、自动发布管理面板实时监控每次运行。

### 2.6 内容级语言全链路

原生支持四种内容语言风格：**简体中文 / 港式繁体 / 粤语口语点缀 / 英文**。从记者偏好语言 → AI prompt 指令注入 → 多平台输出形成端到端可配置链路——**同一篇稿件可同时产出"港式繁体书面语"和"粤语口语点缀"两个版本**，每种风格都有独立的 prompt 调优。对于面向多地区华文受众的媒体机构，这是直接的内容覆盖能力。

### 2.7 支撑功能

- **三级角色权限**（记者/编辑/管理员），所有接口受 JWT 守卫保护；
- **邮件通知 + 审计日志**：审核流转、发布成败、关键异常全程可追溯；
- **按量计费系统**：AI 调用、图片生成、数据抓取按量计费并记录余额、幂等防重——**成本完全透明可控**，决策者可精确掌握每一笔 AI 开销。

## 三、技术创新

CMS-NG 采用 2026 年前沿的全栈技术选型。每一项架构决策都对应一个真实业务痛点——技术在这里是**业务弹性的支撑**，而非炫技。

### 3.1 技术栈

| 层 | 技术选型 |
|---|---|
| **前端** | Next.js 16（App Router）· React 19 · TypeScript · Tailwind CSS v4 · TipTap 富文本 · Zustand 状态管理 |
| **后端** | NestJS 11 · Express · Prisma 6 ORM · MySQL 8 · Redis（ioredis）· JWT（passport-jwt）· @nestjs/schedule |
| **AI** | DeepSeek（默认）/ Kimi / OpenAI（OpenAI 兼容协议 + Provider 抽象层）· Tavily 联网搜索 · 字节火山引擎 Seedream 配图 |
| **工程化** | npm workspaces · Turbo 单仓编排 · Docker Compose · 一键部署脚本 |
| **共享层** | @cms-ng/shared——前后端共享枚举与接口 |

### 3.2 五项关键架构创新

**1. Provider-agnostic 的 AI 抽象层**
`AIService` 不直接调用任何 LLM，而是通过依赖注入容器注入 `ChatCompletionProvider` 接口实现。DeepSeek / Kimi / OpenAI 三个供应商均继承自 `OpenAICompatibleProvider` 抽象基类——任何兼容 OpenAI 协议的模型都能**在不改业务代码的前提下**通过环境变量切换。配合 Tool Registry 模式，新增"联网搜索"或"知识库检索"工具只需实现接口并注册。**业务价值**：不被任何单一模型供应商绑定，模型迭代时零迁移成本。

**2. 平台适配器 + 独立 prompt 编排**
`PlatformRegistry` 注册表 + 工厂模式：每个适配器独立实现接口、拥有独立 prompt 模板与后处理流水线；`PlatformPublish` 表为每平台保存独立版本。**业务价值**：内容本体与平台表达解耦，新增平台只需新增适配器，不影响已有平台。

**3. Pipeline Step 模式 + 失败隔离**
全自动管道每一步实现统一 `PipelineStep` 接口，失败只影响当前文章不阻断批次，失败信息持久化支持单篇重试。**业务价值**：分布式事务思路在内容生产场景的落地——局部失败不拖垮全局，"全自动"不等于"不可控"。

**4. Monorepo 工程化与共享类型层**
前后端共享 `@cms-ng/shared` 包，所有枚举（UserRole/ArticleStatus/Platform/ContentLanguage 等）和核心接口只在 shared 中定义一次，前后端各自消费。**业务价值**：从源头消除"枚举漂移"与"接口不一致"的常见 bug，降低跨端协作成本。

**5. ContentLanguage 贯穿全栈**
从数据库 schema、Article 实体、shared 包、prompt 模板到平台适配器端到端贯穿。**业务价值**：多语种内容能力无需重写模型调用代码即可配置，直接支撑多地区华文受众覆盖。

### 3.3 运维与可靠性

- **Redis fail-open**：缓存层不可用时不崩溃，仅降级告警；
- **Boot 时环境变量校验**：必需变量缺失即快速失败，避免运行时神秘崩溃；
- **七步一键生产发布**：前置检查 → 构建 → 停旧 → 启动 → 迁移 → 健康检查 → admin 初始化，单条命令完成生产更新；
- **MySQL 作为唯一真相源**：Kill Switch 等关键状态直读数据库，不依赖缓存，杜绝状态不一致；
- **外部中间件策略**：不与具体云厂商绑定，同一份部署既可跑在阿里云也可跑在 AWS；
- **地域感知代理开关**：环境变量即可在"大陆开发（走代理）"与"海外生产（直连）"间切换，业务代码零修改。

## 总结

CMS-NG 的产品哲学可以概括为三句话：

1. **AI 是记者的"协作伙伴"而不是"替代者"**——14 项 AI 原子能力 + 9 状态稿件生命周期，让协作发生在最需要协作的环节；
2. **AI 自治是"人力的补充"而不是"风险的来源"**——自动管道配套四道兜底，让"全自动"不等于"不可控"；
3. **技术架构服务于业务弹性**——Provider-agnostic、Platform-agnostic、Region-aware、Monorepo 共享类型，每一条架构决策都对应一个真实业务痛点。

对于希望以最小人力成本覆盖多平台高频更新的媒体机构，CMS-NG 提供了一条**经过工程化验证**、**可平滑演进**、**风险可控**的智能化升级路径——它不是市面上"又一个 AI 写作工具"，而是为媒体编辑部量身定制的**完整内容生产操作系统**。
