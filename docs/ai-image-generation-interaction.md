## 2. 状态设计

### 2.1 空态 (Empty State)

**触发条件**：首次打开面板，或点击"重新分析"后重置。

**视觉设计**：
- 中央大图标：📷 或 🎨（48px，灰色）
- 标题："AI 智能配图"
- 说明文字："基于文章内容，AI 将自动推荐 1-5 张配图方案，涵盖封面图、正文插图和多平台封面。"
- 主按钮："🔄 AI 分析配图需求"（蓝色主按钮）
- 次要提示："分析预计需要 3-5 秒"

**交互**：
- 点击主按钮 → 进入 ANALYZING 状态
- 按钮点击后 disabled，显示 loading spinner

### 2.2 分析中态 (Analyzing State)

**视觉设计**：
- 中央 loading 动画（脉冲圆点或骨架屏）
- 文字："AI 正在分析文章内容..."
- 子文字轮播：
  - "正在提取文章主题..."
  - "正在识别关键场景..."
  - "正在匹配最佳配图位置..."
  - "正在优化生成提示词..."
- 预计时间："预计还需 3 秒"
- 取消按钮："取消分析"

**交互**：
- 点击"取消" → 回到 EMPTY 状态
- 分析完成自动进入 RECOMMENDATIONS 状态
- 分析失败显示错误提示 + "重试"按钮

**推荐卡片设计**：

| 元素 | 样式 | 说明 |
|------|------|------|
| 勾选框 | 左侧 checkbox | 默认全选，用户可取消 |
| 用途标签 | 圆角标签，颜色区分 | 封面=蓝色 🔵 正文=绿色 🟢 平台=橙色 🟠 |
| Prompt 预览 | 2 行截断，灰色文字 | 完整 prompt 可展开 |
| 尺寸 | 小字标签 | 2K/3K/4K 或具体尺寸 |
| 推荐理由 | 带 💡 图标 | 解释为什么需要这张配图 |
| 编辑按钮 | 文字按钮"编辑" | 展开 prompt 编辑区 |

**Prompt 编辑展开态**：
```
┌─────────────────────────────────────────┐
│ ☑️ [封面] 文章封面图                      │
│                                         │
│ ┌─────────────────────────────────────┐ │
│ │ Prompt（245/300字）                  │ │
│ │ ┌─────────────────────────────────┐ │ │
│ │ │香港维多利亚港夜景，现代新闻摄影  │ │ │
│ │ │风格，蓝金色调，大气磅礴...        │ │ │
│ │ └─────────────────────────────────┘ │ │
│ │ [💡 优化建议: 增加"高楼林立"增强都市感]│ │
│ └─────────────────────────────────────┘ │
│                                         │
│ 尺寸: [3K ▼]  2K | 3K | 4K             │
│                                         │
└─────────────────────────────────────────┘
```

**交互**：
- 勾选/取消勾选：实时更新底部"已选择"计数
- 点击"编辑"：展开 prompt 编辑区，textarea 可编辑
- prompt 编辑：实时字数统计，超过 300 字红色警告
- 尺寸下拉：2K / 3K / 4K 三档可选
- 点击"生成配图 (N)"：进入 GENERATING 状态

## 3. 封面图设置交互

### 3.1 文章编辑页封面图区域

**默认状态（无封面）**：
```
┌─────────────────────────────────────────┐
│ 标题: [输入框]                           │
├─────────────────────────────────────────┤
│ ┌─────────────┐                         │
│ │             │  副标题: [输入框]        │
│ │   [占位图   │                         │
│ │    相机图标]│                         │
│ │             │                         │
│ └─────────────┘                         │
│ [📷 AI生成封面] 或 [📤 上传封面]        │
└─────────────────────────────────────────┘
```

**有封面状态**：
```
┌─────────────────────────────────────────┐
│ 标题: [输入框]                           │
├─────────────────────────────────────────┤
│ ┌─────────────┐                         │
│ │             │  副标题: [输入框]        │
│ │ [封面图     │                         │
│ │  缩略图]    │                         │
│ │             │                         │
│ └─────────────┘                         │
│ [🔄 更换封面] [👁️ 预览] [🗑️ 移除]      │
└─────────────────────────────────────────┘
```

**交互**：
- hover 封面图区域 → 显示半透明遮罩 + "更换封面"按钮
- 点击"AI生成封面" → 打开 AI 配图面板并自动触发封面图分析
- 点击"更换封面" → 打开 AI 配图面板（过滤显示用途为 COVER 的图片）
- 点击"移除" → 确认弹窗 → 清空 Article.coverImage

### 3.2 从配图面板设为封面

**流程**：
1. 在结果态中，点击某张图片的"设为封面"按钮
2. 按钮立即变为 "✅ 已是封面"（disabled 态）
3. 如果之前有封面，旧封面卡片恢复为可用状态
4. 文章编辑页封面图区域实时更新缩略图
5. Toast："封面已更新"

**场景**：记者生成了多张平台 OG 图，想一次性分配到对应平台。

**交互**：
1. 在配图面板结果态中，点击某张 PLATFORM_OG 图片的"分配平台"
2. 弹出平台选择器（只显示 supportsImages=true 的平台）
3. 选择平台后，自动关联到该平台封面
4. 如果该平台已有封面，提示"是否替换现有封面？"
5. 渠道分发面板实时更新

## 5. 正文插图插入交互

### 5.1 从配图面板插入

**流程**：
1. 在结果态中，点击 INLINE 类型图片的"插入正文"按钮
2. 弹出段落选择器模态框

```
┌─────────────────────────────────────────┐
│ 选择插入位置                      [×]   │
├─────────────────────────────────────────┤
│                                         │
│  文章段落列表:                          │
│                                         │
│  ○ 文章开头（标题之后）                 │
│  ● 第1段之后  ← 默认选中               │
│  ○ 第2段之后                           │
│  ○ 第3段之后                           │
│  ○ ...                                 │
│  ○ 文章末尾                            │
│                                         │
│  [取消]              [确认插入]         │
│                                         │
└─────────────────────────────────────────┘
```

3. 选择段落位置 → 点击"确认插入"
4. TipTap 编辑器在指定位置插入图片
5. 配图面板中该图片卡片状态变为"已插入 ✅"
6. Toast："图片已插入第 N 段之后"

### 5.2 从编辑器工具栏插入

**流程**：
1. 在 TipTap 编辑器中，点击工具栏"📷 插入图片"按钮
2. 弹出图片选择器

```

│ 插入图片                          [×]   │

│  │[封面]│ │[正文]│ │[FB]  │            │
│  │预览1 │ │预览2 │ │预览3 │            │
│  └──────┘ └──────┘ └──────┘            │

│                                         │
└─────────────────────────────────────────┘
```

3. 选择已生成的图片 → 确认插入
4. 或直接上传本地图片（使用传统 upload 方式）

## 6. 错误与引导设计

### 6.1 错误提示规范

| 错误场景 | 提示类型 | 标题 | 内容 | 操作 |
|----------|----------|------|------|------|

## 7. 响应式设计

### 7.1 桌面端（≥1280px）

- 配图面板：右侧抽屉，宽度 480px
- 推荐卡片：单列列表，卡片宽度 100%
- 结果网格：2 列图片预览

### 7.2 平板端（768px-1279px）

- 配图面板：底部抽屉，高度 60vh
- 推荐卡片：单列列表
- 结果网格：2 列图片预览

### 7.3 移动端（<768px）

- 配图面板：全屏模态
- 推荐卡片：单列列表，卡片垂直堆叠
- 结果网格：单列图片预览
- 段落选择器：底部 action sheet

## 9. 无障碍设计

- 所有图片预览提供 alt 文字（AI 自动生成或记者填写）
- 勾选框、按钮支持键盘焦点和 Enter/Space 操作
- 进度条使用 ARIA progressbar 角色
- 错误提示使用 ARIA alert 角色，屏幕阅读器自动播报
- 颜色不仅是唯一信息载体（勾选状态同时有文字和图标）
- 支持高对比度模式
