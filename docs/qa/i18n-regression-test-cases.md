## 1. 测试范围与策略

### 1.1 测试范围

| 模块 | 覆盖功能 |
|------|----------|
| 选题 (Story) | 创建、更新、详情加载、列表展示、语言持久化 |
| 稿件 (Article) | 创建、更新、详情加载、列表展示、语言持久化 |
| 用户 (User) | 偏好设置、默认值读取 |
| AI 生成 | 12 项 AI 功能的语言参数传递与输出验证 |
| 前端交互 | 语言选择器、保存流程、刷新恢复 |

### 1.2 测试策略

- **前置条件**：每个测试用例执行前，确保数据库已应用最新 migration，前端已重新构建
- **测试数据**：准备 3 个测试用户（REPORTER-A 偏好简体中文、REPORTER-B 偏好英文、REPORTER-C 未设置偏好）
- **验证方式**：前端 UI 验证 + API 接口验证 + 数据库字段验证
- **通过标准**：所有 P0/P1 用例 100% 通过，P2 用例无阻塞级缺陷
